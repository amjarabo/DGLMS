Users Involved & When
PersonaProfile / PSGWhen They Log InWhat They Do
Sys AdminSystem AdministratorAll setup & verification stepsCreates/verifies GL-Application Reviewer PSG. Updates Account Team Role picklist (Reviewer → Application Reviewer). Migrates existing records. Assigns PSG to test user.
App ReviewerStandard + GL-Application Reviewer PSGAccess verificationLogs in after PSG assigned. Navigates to reviewer-gated areas. Confirms access works. Cannot access Setup or admin features.
Non-privileged userStandard / no PSGNegative / security testsAttempts Application Reviewer actions — blocked with Insufficient Privileges.

  • GL-Application Reviewer PSG exists in Setup → Permission Sets
  • PSG contains correct underlying Permission Sets per security matrix (confirm with the development team)
  • Standard-profile test user ready to receive PSG assignment
Verify PSG exists
SELECT Id, Label, DeveloperName FROM PermissionSetGroup WHERE DeveloperName = 'GL_Application_Reviewer'
Preconditions — Account Team Role
  • Account Team Role picklist contains "Application Reviewer" as active value
  • Old "Reviewer" picklist value deactivated or removed
  • All pre-existing AccountTeamMember records with role "Reviewer" migrated
  • Test Account with Account Team related list available
Verify no old "Reviewer" records — expected: 0 rows
SELECT Id, TeamMemberRole FROM AccountTeamMember WHERE TeamMemberRole = 'Reviewer'
Security Tests
  • User WITHOUT PSG → reviewer actions blocked → "Insufficient Privileges"
  • User WITH PSG + standard profile → cannot access Setup / admin features
  • PSG is additive only — does not elevate to System Admin
Negative Tests
  • Account Team Role dropdown does NOT show "Reviewer"
  • Duplicate PSG creation attempt → blocked by Salesforce
  • SOQL for old "Reviewer" role returns 0 rows
